Aliases

Git aliases are shortmands for lazy writers, with these features, you will use git more efficiently, let's see how to add and use it.

Initialize git to your repository

To go to your repository, write the next line on command prompt (cmd):

cd C:\your_repository

and click enter.

Now, to create .git hidden folder typing:

git init

Well done, you have created a new git to your repo!

Configure your first alias

Writing git commands is a boring task, we can continue using normal commands or we can add aliases to our global git conif.

Now is the moment to add these features, at this point I guess you know several git commands, then, let's create a new alias to perform our workspace! The following code is the method that we will use to add it.

git config --global alias.s status

Now, check what can do this config, type:

git s

Console wil print this text:

On branch main

Your branch is up to date with 'origin/main'.

...

Right! You used git status just writing git s

You can add aliases with more than one word like this:

git config --global alias.cob "checkout -b"

Perfect! Now we can use more aliases, there is a list that you can use to be more efficient using git.

List of aliases

alias.s status

alias.b branch

alias.co checkout

alias.cob checkout -b

alias.aa add .

alias.cm commit -m
